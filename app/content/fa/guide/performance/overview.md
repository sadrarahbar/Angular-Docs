با روش‌های مختلفی آشنا شوید که می‌توانید performance برنامه‌ی خود را با rendering strategyهای گوناگون بهینه کنید.
</docs-decorative-header>

یکی از اولویت‌های اصلی هر developer این است که برنامه‌اش تا جای ممکن performant باشد. این راهنماها کمک می‌کنند با استفاده از rendering strategyهای مختلف، best practiceهای ساخت برنامه‌های performant را دنبال کنید.

| نوع راهنما | توضیح |
| :--------- | :---- |
| [Server and hybrid rendering](/guide/ssr) | یاد بگیرید چطور با render کردن صفحه‌ها روی server، زمان load را بهتر کنید. |
| [Hydration](/guide/hydration) | فرایندی برای بهتر کردن performance برنامه با restore کردن state آن بعد از server-side rendering و reuse کردن ساختار DOM موجود تا جای ممکن. |
| [Incremental Hydration](/guide/incremental-hydration) | Incremental hydration نوع پیشرفته‌ای از hydration است که می‌تواند بخش‌هایی از برنامه را dehydrated نگه دارد و hydration آن بخش‌ها را هر وقت لازم شدند به‌صورت incremental trigger کند. |
