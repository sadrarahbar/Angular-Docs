# بهینه‌سازی کارایی زمان اجرا

رندر سریع برای Angular حیاتی است و ما framework را با درنظرگرفتن بهینه‌سازی‌های فراوان ساخته‌ایم تا به شما در توسعهٔ برنامه‌های کارآمد کمک کند. برای درک بهتر کارایی برنامه، [Angular DevTools](tools/devtools) و یک [راهنمای ویدیویی](https://www.youtube.com/watch?v=FjyX_hkscII) دربارهٔ استفاده از Chrome DevTools برای profiling ارائه کرده‌ایم. در این بخش، رایج‌ترین روش‌های بهینه‌سازی کارایی را بررسی می‌کنیم.

**Change detection** فرایندی است که Angular از طریق آن بررسی می‌کند آیا state برنامهٔ شما تغییر کرده و آیا لازم است بخشی از DOM به‌روزرسانی شود یا نه. در سطحی کلی، Angular componentهای شما را از بالا به پایین پیمایش می‌کند و به‌دنبال تغییرات می‌گردد. Angular سازوکار change detection خود را به‌صورت دوره‌ای اجرا می‌کند تا تغییرات data model در view برنامه بازتاب پیدا کنند. Change detection می‌تواند به‌صورت دستی یا از طریق یک event ناهمگام (برای مثال، تعامل کاربر یا تکمیل یک XMLHttpRequest) فعال شود.

Change detection بسیار بهینه و کارآمد است، اما اگر برنامه آن را بیش‌ازحد اجرا کند، همچنان می‌تواند باعث کندی شود.

در این راهنما یاد می‌گیرید چگونه با نادیده‌گرفتن بخش‌هایی از برنامه و اجرای change detection فقط در زمان لازم، این سازوکار را کنترل و بهینه کنید.
