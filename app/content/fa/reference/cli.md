# مرجع CLI

| فرمان                              | نام مستعار | توضیحات                                                                                              |
| :--------------------------------- | :--------- | :--------------------------------------------------------------------------------------------------- |
| [`add`](cli/add)                   |            | پشتیبانی از یک کتابخانه خارجی را به پروژه اضافه می‌کند.                                              |
| [`analytics`](cli/analytics)       |            | جمع‌آوری معیارهای استفاده از Angular CLI را پیکربندی می‌کند.                                         |
| [`build`](cli/build)               | `b`        | application یا کتابخانه انگولار را در دایرکتوری خروجی به نام dist/ در مسیر خروجی داده‌شده کامپایل می‌کند. |
| [`cache`](cli/cache)               |            | cache پایدار روی دیسک را پیکربندی و آمار cache را دریافت می‌کند.                                     |
| [`completion`](cli/completion)     |            | تکمیل خودکار Angular CLI را برای terminal راه‌اندازی می‌کند.                                         |
| [`config`](cli/config)             |            | مقادیر پیکربندی انگولار را در فایل angular.json مربوط به workspace دریافت یا تنظیم می‌کند.            |
| [`deploy`](cli/deploy)             |            | builder مربوط به deploy را برای پروژه مشخص یا پروژه پیش‌فرض workspace فراخوانی می‌کند.                |
| [`e2e`](cli/e2e)                   | `e`        | application انگولار را build و serve می‌کند، سپس testهای end-to-end را اجرا می‌کند.                   |
| [`extract-i18n`](cli/extract-i18n) |            | پیام‌های i18n را از کد منبع استخراج می‌کند.                                                          |
| [`generate`](cli/generate)         | `g`        | فایل‌ها را بر اساس یک شماتیک تولید یا اصلاح می‌کند.                                                   |
| [`lint`](cli/lint)                 |            | ابزارهای lint را روی کد application انگولار در پوشه پروژه داده‌شده اجرا می‌کند.                       |
| [`new`](cli/new)                   | `n`        | یک workspace جدید انگولار ایجاد می‌کند.                                                              |
| [`run`](cli/run)                   |            | target مربوط به Architect را با پیکربندی اختیاری builder سفارشی تعریف‌شده در پروژه اجرا می‌کند.       |
| [`serve`](cli/serve)               | `s`, `dev` | application را build و serve می‌کند و با تغییر فایل‌ها آن را دوباره می‌سازد.                          |
| [`test`](cli/test)                 | `t`        | unit testهای پروژه را اجرا می‌کند.                                                                    |
| [`update`](cli/update)             |            | workspace و dependencyهای آن را به‌روزرسانی می‌کند. [راهنمای به‌روزرسانی](/update-guide) را ببینید.  |
| [`version`](cli/version)           | `v`        | نسخه Angular CLI را نمایش می‌دهد.                                                                     |
