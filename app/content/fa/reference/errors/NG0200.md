# Circular Dependency در DI

<docs-video src="https://www.youtube.com/embed/CpLOm4o_FzM"/>

dependency چرخه‌ای زمانی وجود دارد که [dependency یک service](guide/di/hierarchical-dependency-injection) به صورت مستقیم یا غیرمستقیم به خود همان service وابسته باشد. برای مثال، اگر `UserService` به `EmployeeService` وابسته باشد و `EmployeeService` هم به `UserService` وابسته باشد. Angular باید برای ساخت `UserService`، ابتدا `EmployeeService` را instantiate کند؛ در حالی که آن هم به خود `UserService` وابسته است.

## debug کردن error

از call stack استفاده کنید تا مشخص شود dependency چرخه‌ای کجا وجود دارد. با [map کردن](guide/di/di-in-action) dependencyهای کامپوننت، module یا service و شناسایی loop ایجادکننده مشکل، می‌توانید ببینید آیا dependencyهای child به فایل اصلی وابسته‌اند یا نه.

برای رفع این error، این loop یا circle مربوط به dependency را بشکنید. رایج‌ترین معنی‌اش این است که dependencyها را حذف یا refactor کنید تا به یکدیگر وابسته نباشند.
