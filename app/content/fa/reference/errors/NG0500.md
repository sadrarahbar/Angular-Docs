# ناهماهنگی node در Hydration

این error یعنی در طول فرآیند hydration، Angular انتظار داشته ساختار DOM همان‌طور باشد که هنگام server-side rendering render و annotate شده بود. اما در client، DOM tree با DOM tree render شده روی server متفاوت بوده است.

این error معمولاً به خاطر دستکاری مستقیم DOM با APIهای native مرورگر رخ می‌دهد؛ دستکاری‌ای که ساختار DOM را خارج از چیزی که Angular تولید کرده تغییر می‌دهد. همچنین اگر از `innerHTML` یا `outerHTML` برای تنظیم محتوای HTML استفاده کنید رخ می‌دهد، چون آن‌ها هم ساختار DOM را خارج از چیزی که Angular تولید کرده تغییر می‌دهند. می‌توانید این مشکل را با refactor کردن کامپوننت برای استفاده از APIهای native Angular به جای APIهای native مرورگر حل کنید. اگر این ممکن نیست، می‌توانید attribute مربوط به `ngSkipHydration` را به host node کامپوننت خود اضافه کنید تا hydration برای آن کامپوننت و فرزندانش غیرفعال شود. `ngSkipHydration` فقط باید به عنوان آخرین راه‌حل استفاده شود و باید آن را bugای در نظر گرفت که نیاز به رفع دارد.

اطلاعات بیشتر درباره hydration را می‌توانید در [این راهنما](guide/hydration) پیدا کنید.

مثال زیر error را trigger می‌کند.

```angular-ts
@Component({
  selector: 'app-example',
  template: '<div><span>world</span></div>',
})
export class Example {
  hostElement = inject(ElementRef).nativeElement;

  ngOnInit() {
    // Create a new <p> element with the `Hello` text inside
    const newNode = document.createElement('p');
    newNode.innerHTML = 'Hello';

    // Insert the <p> before the first element. Since Angular has no information
    // about the <p> element, it will be looking for the <div> element at the first
    // element position instead. As a result, a hydration mismatch error would be
    // thrown. Instead, update component's template to create the <p> element.
    this.hostElement.insertBefore(newNode, this.hostElement.firstChild);
  }
}
```

## debug کردن error

پیام error در developer console باید اطلاعاتی درباره بخش مشخصی از ساختار DOM برنامه که باعث مشکل شده داشته باشد. آن بخش از برنامه را برای errorهای مرتبط با hydration، مثل دستکاری مستقیم DOM با APIهای native، بررسی کنید.

بررسی کنید template شما ساختار HTML معتبر داشته باشد. برای اطلاعات بیشتر، [راهنمای hydration](guide/hydration#valid-html-structure) را ببینید.

می‌توانید به عنوان workaround احتمالی، attribute مربوط به `ngSkipHydration` را به host node کامپوننت خود اضافه کنید.

اگر برنامه شما در development environment کار می‌کند، اما در buildهای production errorهای hydration می‌بینید، مطمئن شوید HTML تولیدشده‌ای که به مرورگر تحویل داده می‌شود شامل comment nodeهای تولیدشده توسط Angular هنگام rendering روی server باشد. Angular runtime از این comment nodeها به عنوان anchor برای view containerها، با hydration و بدون hydration، استفاده می‌کند و فرآیند hydration انتظار دارد آن‌ها در locationهای اصلی خود وجود داشته باشند. اگر logic سفارشی برای حذف comment nodeها از HTML تولیدشده توسط server-side rendering دارید یا CDN خود را طوری پیکربندی کرده‌اید که قبل از serve کردن محتوا آن‌ها را حذف کند، حذف comment nodeها را غیرفعال کنید و بررسی کنید آیا errorهای hydration از بین رفته‌اند یا نه.
