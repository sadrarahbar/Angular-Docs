# flag مربوط به skip hydration روی node نامعتبر اعمال شده است

این error زمانی رخ می‌دهد که attribute مربوط به `ngSkipHydration` به یک DOM node نامناسب اضافه شده باشد. attribute مربوط به `ngSkipHydration` فقط می‌تواند روی host nodeهای کامپوننت اعمال شود؛ یا مستقیماً در template یا از طریق host binding. این attribute نمی‌تواند روی DOM nodeهای دیگر اعمال شود و اگر چنین کاری انجام شود، به جز ایجاد این error اثری نخواهد داشت.

اطلاعات بیشتر درباره hydration را می‌توانید در [این راهنما](guide/hydration) پیدا کنید.

مثال‌های زیر این error را trigger می‌کنند.

## مثال 1

در این مثال، attribute مربوط به `ngSkipHydration` با استفاده از host bindingهای یک directive روی یک `<div>` اعمال شده است. چون `<div>` به عنوان host node کامپوننت عمل نمی‌کند، Angular یک error throw می‌کند.

```angular-ts
@Directive({
  selector: '[dir]',
  host: {ngSkipHydration: 'true'},
})
class Dir {}

@Component({
  selector: 'app',
  imports: [Dir],
  template: ` <div dir></div> `,
})
class Simple {}
```

## مثال 2

در این مثال، `ngSkipHydration` به عنوان یک attribute از طریق template روی یک `<div>` اعمال شده است. چون `<div>` به عنوان host node کامپوننت عمل نمی‌کند، Angular یک error throw می‌کند.

```angular-ts
@Component({
  selector: 'app',
  template: ` <div ngSkipHydration></div> `,
})
class Simple {}
```

## debug کردن error

attribute مربوط به `ngSkipHydration` را از هر DOM node نامعتبر حذف کنید. به عنوان راه جایگزین، attribute مربوط به `ngSkipHydration` را به host node کامپوننت منتقل کنید؛ یا در template یا از طریق host binding.
