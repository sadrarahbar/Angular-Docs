# محتوای HTML بعد از server-side rendering تغییر کرده است

Angular زمانی این error را throw می‌کند که تشخیص دهد محتوای تولیدشده توسط server-side rendering \(SSR\) بعد از rendering تغییر کرده است. فرآیند hydration به دست‌نخورده ماندن محتوا بعد از SSR وابسته است؛ این شامل whitespaceها و comment nodeها هم می‌شود. آن whitespaceها و comment nodeها باید در HTML تولیدشده توسط فرآیند SSR حفظ شوند. در راهنمای Hydration بیشتر یاد بگیرید.

بعضی providerهای CDN یک قابلیت built-in برای حذف whitespaceها و comment nodeها از HTML به عنوان optimization دارند. بررسی کنید آیا چنین گزینه‌ای در پیکربندی CDN وجود دارد و آن را خاموش کنید.

اگر از post-processing سفارشی روی HTML تولیدشده توسط SSR، مثلاً به عنوان یک build step، استفاده می‌کنید، مطمئن شوید این فرآیند whitespaceها و comment nodeها را حذف نمی‌کند.
