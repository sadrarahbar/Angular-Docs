# اطلاعات hydration در response server وجود ندارد

این error زمانی رخ می‌دهد که hydration روی client فعال است، اما response مربوط به server شامل اطلاعات serialized ویژه‌ای درباره برنامه نیست که logic مربوط به hydration به آن وابسته است.

این می‌تواند زمانی رخ دهد که تابع `provideClientHydration()` در بخش client مربوط به پیکربندی برنامه قرار داده شده، اما در بخش server پیکربندی وجود ندارد.

در برنامه‌هایی با ساختار پیش‌فرض پروژه، که با `ng new` ساخته شده‌اند، فراخوانی `provideClientHydration()` یا به آرایه `providers` در `AppModule` اصلی اضافه می‌شود، که داخل server module import شده، یا در مجموعه‌ای از providerها قرار می‌گیرد که هم در پیکربندی client و هم server استفاده می‌شوند.

اگر setup سفارشی دارید که در آن پیکربندی client و server مستقل هستند و مجموعه مشترکی از providerها را share نمی‌کنند، مطمئن شوید `provideClientHydration()` در مجموعه providerهایی که برای bootstrap کردن برنامه روی server استفاده می‌شوند هم قرار داشته باشد.

اطلاعات بیشتر درباره hydration را می‌توانید در [این راهنما](guide/hydration) پیدا کنید.

## debug کردن error

بررسی کنید که فراخوانی `provideClientHydration()` در مجموعه providerهایی قرار داشته باشد که برای bootstrap کردن برنامه روی server استفاده می‌شوند.
