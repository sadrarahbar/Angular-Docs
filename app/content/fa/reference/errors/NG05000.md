# Hydration با instance پشتیبانی‌نشده Zone.js

این warning یعنی hydration برای برنامه‌ای فعال شده که برای استفاده از نسخه‌ای پشتیبانی‌نشده از Zone.js پیکربندی شده است: یا یک نسخه سفارشی، یا یک نسخه "noop"؛ برای اطلاعات بیشتر [اینجا](api/core/BootstrapOptions#ngZone) را ببینید.

Hydration به signalای از Zone.js وابسته است؛ زمانی که Zone.js داخل برنامه stable می‌شود، Angular می‌تواند فرآیند serialization را روی server یا cleanup بعد از hydration را روی client شروع کند، تا DOM nodeهایی را که unclaimed مانده‌اند حذف کند.

ارائه یک implementation سفارشی یا "noop" از Zone.js ممکن است زمان‌بندی متفاوتی برای event مربوط به "stable" ایجاد کند و در نتیجه serialization یا cleanup خیلی زود یا خیلی دیر trigger شود. این هنوز یک پیکربندی کاملاً پشتیبانی‌شده نیست.

اگر از implementation سفارشی Zone.js استفاده می‌کنید، مطمئن شوید event مربوط به "onStable" در زمان درست emit می‌شود و باعث رفتار نادرست برنامه با hydration نمی‌شود.

اطلاعات بیشتر درباره hydration را می‌توانید در [راهنمای hydration](guide/hydration) پیدا کنید.
