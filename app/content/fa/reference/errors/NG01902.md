# field جداافتاده در signal form

این خطا نشان می‌دهد یک instance از `Field` دیگر به ساختار فعلی model متصل نیست.
signal formهای Angular زمانی `NG01902` ایجاد می‌کنند که path مربوط به field که قبلاً وجود داشت، دیگر به‌عنوان همان property در object والد resolve نشود.

یکی از عوامل رایج، تنظیم ناخواسته مقدار field روی `undefined` است.
در signal form،‏ `undefined` یعنی «field وجود ندارد»؛ بنابراین field موجود ممکن است جداافتاده شود.

## چرا این اتفاق رخ می‌دهد؟

signal formها instanceهای field را با شکل model هماهنگ نگه می‌دارند.
اگر پس از تغییر شکل model، کد همچنان به instance قدیمی field ارجاع دهد، Angular این ناهماهنگی را تشخیص داده و خطا ایجاد می‌کند.

این اتفاق ممکن است زمانی رخ دهد که:

- property از object مربوط به model حذف شود.
- property به‌جای باقی‌ماندن در object، مقدار `undefined` بگیرد.
- کد در طول به‌روزرسانی‌های ساختاری یک reference قدیمی field را نگه دارد.

## چگونه آن را رفع کنیم؟

- مقدار `undefined` را در fieldهای model ننویسید. وقتی field باید همچنان وجود داشته باشد، از `null` یا مقداری صریح استفاده کنید.
- هنگام استفاده از یک field، شکل model را پایدار نگه دارید.
- پس از به‌روزرسانی ساختاری، به‌جای استفاده دوباره از reference قدیمی، fieldها را از درخت فعلی form بخوانید.

برای راهنمای طراحی model، بخش [پرهیز از `undefined`](/guide/forms/signals/model-design#avoid-undefined) را ببینید.
