# تابع `inject()` باید از injection context فراخوانی شود

این خطا زمانی ظاهر می‌شود که تابع [`inject`](api/core/inject) را خارج از [injection context](guide/di/dependency-injection-context) مجاز به‌کار ببرید. injection context هنگام ساخت و مقداردهی اولیه class در دسترس است. همچنین برای توابعی که
همراه با `runInInjectionContext` استفاده می‌شوند در دسترس است.

در عمل فراخوانی `inject()` در constructor، پارامتر constructor و field initializer مجاز است:

```typescript
@Service()
export class Car {
  radio: Radio | undefined;

  // OK: field initializer
  spareTyre = inject(Tyre);

  constructor() {
    // OK: constructor body
    this.radio = inject(Radio);
  }
}
```

فراخوانی [`inject`](api/core/inject) از factory مربوط به provider نیز مجاز است:

```typescript
providers: [
  {
    provide: Car,
    useFactory: () => {
      // OK: a class factory
      const engine = inject(Engine);
      return new Car(engine);
    },
  },
];
```

فراخوانی تابع [`inject`](api/core/inject) خارج از ساخت class یا `runInInjectionContext` به خطا منجر می‌شود. به‌طور خاص، فراخوانی `inject()` پس از ساخته‌شدن instance مربوط به class و در methodها، از جمله lifecycle hookها، مجاز نیست:

```typescript
@Component({ ... })
export class Car {
  ngOnInit() {
    // ERROR: too late, the component instance was already created
    const engine = inject(Engine);
    engine.start();
  }
}
```

## اشکال‌زدایی خطا

در stack trace خطا به عقب حرکت کنید تا محل فراخوانی غیرمجاز `inject()` را پیدا کنید.

برای رفع خطا، فراخوانی [`inject`](api/core/inject) را به محل مجاز، معمولاً constructor یا field initializer مربوط به class، منتقل کنید.

**توجه:** اگر در test context اجرا می‌کنید، `TestBed.runInInjectionContext` باعث موفقیت `inject()` می‌شود.

```typescript
TestBed.runInInjectionContext(() => {
  // ...
});
```
