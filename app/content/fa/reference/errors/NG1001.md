# Argument باید literal باشد

برای سریع‌تر شدن metadata extraction در Angular compiler، decoratorهای `@NgModule`، `@Pipe`، `@Component`، `@Directive` و `@Injectable` فقط object literal را به عنوان argument می‌پذیرند.

این یک [تغییر عمدی در Ivy](https://github.com/angular/angular/issues/30840#issuecomment-498869540) است که requirementهای سخت‌گیرانه‌تری برای argumentهای decoratorها نسبت به View Engine اعمال می‌کند. Ivy به این رویکرد نیاز دارد چون decoratorها را با جابه‌جا کردن expressionها به مکان‌های دیگر در خروجی class compile می‌کند.

## debug کردن error

همه declarationها را از این حالت:

```ts
const moduleDefinition = {…}

@NgModule(moduleDefinition)
export class AppModule {
  constructor() {}
}

```

به داخل decorator منتقل کنید:

```ts

@NgModule({…})
export class AppModule {
  constructor() {}
}

```
