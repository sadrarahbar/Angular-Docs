# مقدار control وجود ندارد

این خطا زمانی رخ می‌دهد که `setValue` را روی `FormGroup` یا `FormArray` فراخوانی کنید، اما مقدار ارائه‌شده entry مربوط به یکی از controlهای ثبت‌شده را نداشته باشد.

`setValue` سخت‌گیرانه است و برای هر control یک مقدار می‌خواهد. اگر فقط برخی controlها را به‌روزرسانی می‌کنید، به‌جای آن از `patchValue` استفاده کنید.

## اشکال‌زدایی خطا

نام control را در پیام خطا پیدا و مطمئن شوید object مربوط به مقدار شامل آن است.

یکی از دلایل رایج، spread کردن objectای است که تمام کلیدها را ندارد:

```typescript
const someValue = {first: 'Nancy'}; // 'last' is missing

form.setValue({...someValue}); // throws NG01002
```

این وضعیت ممکن است زمانی رخ دهد که `someValue` از response یک API، به‌روزرسانی جزئی state یا نوعی که کاملاً با ساختار form مطابقت ندارد آمده باشد. در این موارد کلیدهای مفقود را صریحاً پر کنید یا به `patchValue` تغییر دهید.

### FormGroup

```typescript
const form = new FormGroup({
  first: new FormControl(''),
  last: new FormControl(''),
});

// 'last' is not in the value — throws NG01002
form.setValue({first: 'Nancy'});

// both controls are covered — works fine
form.setValue({first: 'Nancy', last: 'Drew'});
```

### FormArray

```typescript
const formArray = new FormArray([new FormControl(''), new FormControl('')]);

// only one value for two controls — throws NG01002
formArray.setValue(['Nancy']);

// one value per control — works fine
formArray.setValue(['Nancy', 'Drew']);
```
