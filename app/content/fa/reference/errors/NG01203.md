# value accessor وجود ندارد

برای تمام form controlهای سفارشی باید یک value accessor ثبت کنید.

نمونه ارائه یک value accessor:

```typescript
providers: [
  {
    provide: NG_VALUE_ACCESSOR,
    useExisting: forwardRef(() => MyInputField),
    multi: true,
  },
];
```

## اشکال‌زدایی خطا

همان‌طور که گفته شد، انتظار می‌رفت control شما value accessor داشته باشد، اما نداشت. دلایل گوناگونی ممکن است در عمل باعث این اتفاق شوند. برخی مشکلات شناخته‌شده منجر به این خطا عبارت‌اند از:

1. اگر یک form control سفارشی **تعریف کرده‌اید**، آیا value accessor را ارائه داده‌اید؟
1. آیا `ngModel` را روی element بدون مقدار یا **element نامعتبر** مانند `<div [(ngModel)]="foo">` قرار داده‌اید؟
1. آیا از form control سفارشی declarationشده در یک `NgModule` استفاده می‌کنید؟ در این صورت مطمئن شوید `NgModule` را **import** کرده‌اید.
1. آیا `ngModel` را با form control سفارشی شخص ثالث به‌کار می‌برید؟ بررسی کنید آن control یک value accessor ارائه می‌دهد. در غیر این صورت روی element مربوط به control از **`ngDefaultControl`** استفاده کنید.
1. آیا یک form control سفارشی را **تست** می‌کنید؟ مطمئن شوید testbed برای شناخت control پیکربندی شده است. این کار را می‌توانید با `Testbed.configureTestingModule` انجام دهید.
1. آیا از **Nx و Module Federation** همراه webpack استفاده می‌کنید؟ ممکن است `webpack.config.js` برای اشتراک بسته forms به [پیکربندی بیشتر](https://github.com/angular/angular/issues/43821#issuecomment-1054845431) نیاز داشته باشد.
