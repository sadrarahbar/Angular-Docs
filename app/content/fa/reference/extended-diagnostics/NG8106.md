# suffix پشتیبانی نمی‌شود

این diagnostic زمانی را تشخیص می‌دهد که suffixهای `.px`، `.%` و `.em` همراه با attribute binding استفاده شوند.

```html
<img [attr.width.px]="5" />
```

## مشکل این چیست؟

این suffixها فقط برای style bindingها در دسترس هستند. هنگام binding به یک attribute هیچ معنایی ندارند.

## به جای آن چه کار کنم؟

به جای استفاده از suffixهای `.px`، `.%` یا `.em` که فقط در style bindingها پشتیبانی می‌شوند، این مقدار را به value assignment مربوط به binding منتقل کنید.

```html
<img [attr.width]="'5px'" />
```

## الزامات پیکربندی

برای emit شدن هر extended diagnostic باید [`strictTemplates`](tools/cli/template-typecheck#strict-mode) فعال باشد. `suffixNotSupported` فراتر از `strictTemplates` الزام دیگری ندارد.

## اگر نتوانم از این حالت اجتناب کنم چه؟

این diagnostic را می‌توان با ویرایش فایل `tsconfig.json` پروژه غیرفعال کرد:

```json
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "suffixNotSupported": "suppress"
      }
    }
  }
}
```

برای اطلاعات بیشتر، [پیکربندی extended diagnostic](extended-diagnostics#configuration) را ببینید.
